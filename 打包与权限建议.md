# DustDB v1 打包与权限说明

## 1. 最小运行环境

DustDB v1 使用 Python 编写。图形界面使用 Python 自带的 `tkinter`，读取和另存 xlsx 使用 `openpyxl`。

最小环境：

1. Windows 10 或 Windows 11
2. Python 3.10 或更高版本
3. `openpyxl`

环境配置：

```powershell
cd {工具目录}
.\setup_env.ps1
```

源码运行：

```powershell
cd {工具目录}
.\run_tool.ps1
```

## 2. 打包方式

DustDB v1 使用 PyInstaller 打包为单个 exe。

打包命令：

```powershell
cd {工具目录}
.\build_exe.ps1
```

打包结果：

`{工具目录}\dist\DustDB_v1.exe`

打包脚本只打包查询下载工具，不打包原始数据库。运行软件时，用户需要选择汇总表、光谱文件夹和导出位置。

## 3. 数据与软件的关系

DustDB v1 与原始数据库分离：

1. 软件负责读取汇总表、筛选记录、复制光谱文件和另存导出结果。
2. 原始数据库保持独立文件夹结构。
3. 软件更新不影响原始数据。
4. 原始数据更新后，无需重新打包软件。

## 4. 数据库是否打包进软件

当前数据库不打包进 exe。

原因包括：

1. 光谱文件数量较多，完整打包会显著增加软件体积。
2. 打包后的数据更新需要重新构建 exe。
3. exe 内置数据不等同于严格加密。
4. 当前工具的功能定位是查询、筛选和导出，不是封闭式数据授权系统。

## 5. 本地权限说明

Windows 可对文件夹设置读取、写入和修改权限。该权限依赖当前电脑的系统账户。

当数据库位于 U 盘、移动硬盘、压缩包或其他电脑中时，权限继承和生效情况受文件系统、复制方式和目标电脑账户控制。

本地文件权限可用于降低误删、误改风险。若需要限制不同用户的数据访问范围，应采用服务器、局域网数据库、加密容器或数据库账户管理。

## 6. 当前版本边界

DustDB v1 不执行以下操作：

1. 不修改原始汇总表。
2. 不修改原始光谱 txt。
3. 不计算光谱指数。
4. 不编辑数据库结构。
5. 不提供账号登录或分级授权功能。

DustDB v1 执行以下操作：

1. 读取汇总表。
2. 根据字段条件筛选样本。
3. 按 `标识码` 查找光谱 txt。
4. 将筛选结果另存为 CSV 或 XLSX。
5. 将所选光谱复制到导出目录。
6. 生成缺失光谱清单和导出日志。
